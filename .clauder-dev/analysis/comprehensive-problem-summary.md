## 2. 근본 원인 분석

### 원인 1: "수동적 문서 시스템"
```
문서 존재 ≠ 실행 보장
- 읽기만 가능
- 실행 불가능
- 검증 불가능
```

### 원인 2: "토큰 최적화 vs 실효성 충돌"
```
토큰 절약 → 정보 부족 → 효과성 저하
필요 정보 포함 → 토큰 증가 → 비효율
```

### 원인 3: "상태 관리 부재"
```
- 현재 작업 단계 모름
- 이전 작업 기억 못함
- 학습 축적 안 됨
```
